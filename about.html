Branch & Merge test

Use this file to verify making changes on another branch and merging once everything looks correct.

Recommended workflow

  1. Create and switch to a new branch for your change.
  2. Edit files and commit the changes locally.
  3. Push the branch to the remote and open a Pull Request (PR) or merge locally after review.
  4. Once verified, merge into the target branch (e.g., main) and delete the feature branch.

Commands (copy / paste)

# create and switch to a new branch
git checkout -b feature/test-about

# make changes, then stage & commit
git add about.html
git commit -m "Update about.html for branch/merge test"

# push branch to origin
git push -u origin feature/test-about

# after review, merge to main (option A: via PR on remote)
# Option B: merge locally
git checkout main
git pull origin main
git merge --no-ff feature/test-about
git push origin main

# optionally delete the feature branch
git branch -d feature/test-about
git push origin --delete feature/test-about

If you plan to use a GUI (GitHub/GitLab/Bitbucket), open a PR from feature/test-about into main, review, then merge from the web UI.
